## Information Processing (Part 2)

### Goal 1 : Sensory Storage, Working Memory, Long-Term Memory 및 Attention의 역할 [P20, 23, 24]
- **HIP 모델**: Sensory Storage → WM → LTM
- **Sensory Storage**: 주의 선택 전 잔상 유지. Iconic(시각) <1초 / Echoic(청각) 2~4초
- **WM**: **7±2 chunks** (실제 ~4), rehearsal 없으면 **15~30초** 소멸
  - 코드: Visual(공간) / Phonetic(언어) / Semantic(의미), **Dual Coding** = Visual+Phonetic 결합 시 기억↑
- **LTM**: semantic 기반 영구 저장, 용량 무제한, 검색 속도 ∝ association 강도
- **Attention**: Sensory→WM gateway. Maintenance rehearsal(반복→WM 유지) / Elaborative rehearsal(기존 지식 연결→LTM 전달)

### Goal 2 : Chunking으로 WM 용량 한계 극복 [P20, 25]
- **Chunking**: 개별 정보를 LTM 기반 의미 단위로 재코딩. WM 한계는 **chunk 수**에 의해 결정
- 예: 4444555566667777 → 4444 5555 6666 7777 (16→4 chunks), (123)456-7890 (10→3 chunks)
- 전문가 chunk ≠ 초보자 chunk → **사용자 친숙도** 고려 필수

### Goal 3 : Semantic Networks, Schemas, Recognition vs Recall [P21, 27, 28]
- **Semantic Networks**: 의미·연관성 기반 저장, 강한 연관 → 빠른 접근
- **Schemas**: 지식 조직하는 mental framework (예: "File" 메뉴 → Open, Save 자동 연상)
- **Recognition**(단서 있음, 쉬움) > **Recall**(단서 없음, 어려움) → "See & choose" > "Remember & type"
- **설계**: 논리적 그룹화, consistent labeling, memory 외재화(breadcrumbs, visible indicators)

### Goal 4 : Selective / Focused / Divided / Sustained Attention [P21, 29]
- **Selective**: 여러 채널 스캔→단일 과제. Load stress: 채널 수↑→성능↓. 자주 쓰는 display 가까이 배치
- **Focused**: 한 소스 집중, 방해 억제. 실패=distraction interference or hyper-focus로 alarm 놓침
- **Divided**: 2+과제 동시 수행. 동일 자원 경쟁→성능↓↓. **Automaticity**(반복→자동화)로 자원 소모↓
- **Sustained (Vigilance)**: 장기간 드문 신호 탐지. 실패: distraction, tunneling, fatigue

### Goal 5 : Vigilance Decrement 원인 및 대응책 [P21, 29]
- **첫 20~30분** 내 detection 성능 급락. 원인: time-on-task fatigue + low signal probability
- **대응책**: 20~30분 제한 후 휴식 / signal conspicuity↑ / 즉각 KR feedback / artificial signal 삽입 / 공간·시간 불확실성↓ / 환경 최적화 / 충분한 training

### Goal 6 : Wickens' Multiple Resource Theory (MRT) [P21, 30]
- **4 Dimensions**: Input(Visual/Auditory) × Stage(Perceptual-Cognitive/Response) × Code(Spatial/Verbal) × Response(Manual/Vocal)
- 동일 차원 같은 level 공유 → **High Conflict**. 다른 level → Low Conflict
- **Driving 예**: 운전(Visual+Spatial+Manual) + hands-free(Auditory+Verbal+Vocal) = Low conflict / texting(**Visual**+Verbal+**Manual**) = **High conflict** (23x 사고↑)
- 설계: 동시 과제를 다른 resource pool에 배분, 긴급 경고→청각 offload

### Goal 7 : Conceptual / Movement / Spatial / Modality Compatibility [P22, 32, 33]
- **Compatibility**: S-R 관계가 인간 기대(mental model)와 일치하는 정도. 높을수록 mental recoding 제거→RT↓, 오류↓
- **Conceptual**: 기호=개념 일치 (🗑️=삭제, Red=STOP, Green=GO)
- **Movement**: 조작↔반응 방향 일치 (Population Stereotypes). CW=증가, Up/Right=On/Increase
- **Spatial**: 컨트롤-디스플레이 물리적 배치 일치. Isomorphism(공간 동형)이 최적
- **Modality**: 과제↔입출력 양식 적합. Verbal task→Auditory+Vocal / Spatial task→Visual+Manual
- 우선순위: **movement > spatial** (운동 stereotype 의존도 높음)

### Goal 8 : 고령자(~65+) 인지 변화 및 설계 수정 [P22, 33]
- **저하**: General slowing / WM disruption / LTM encoding↓ / Incompatibility 대응↓ / Perceptual deficits
- **유지**: Crystallized intelligence (지식·경험)
- **설계 (Universal Design)**: signal 강화(대비·글꼴↑) / multitasking↓→순차 설계 / compatibility 극대화 / cues·previews / error recovery 지원 / simplicity

### Goal 9 : Mental Workload 정의 및 측정 [P22, 34, 35]
- **Workload = Demand / Capacity**. 역U자 곡선: Underload(지루)→Optimal→Overload(오류↑)
- **측정 4가지**: Primary task(주 과제 성능) / Secondary task(여유 용량) / Physiological(HRV, pupil, P300) / Subjective(NASA-TLX, SWAT)
- **Triangulation**: 여러 카테고리 병용→실제 인지 비용 파악

### Goal 10 : NASA-TLX 6 Subscales [P22, 34]
- **6차원**: Mental Demand / Physical Demand / Temporal Demand / Performance / Effort / Frustration
- **절차**: (1) Pairwise weighting(15쌍, W=0~5) → (2) Raw rating(0~100) → (3) **Score = Σ(W×R) / 15**
- 장점: 간편, 널리 채택. 한계: 사후 측정→memory bias, effort로 성능 유지 시 괴리

### Goal 11 : Human-AI Interaction 과제 [P23, 35, 36]
- **Knowledge Acquisition Bottleneck**: tacit knowledge → explicit rules 변환 시 정보 손실
- **Explainability**: what만이 아닌 **what + why** 설명해야 적절한 신뢰 확보
- **Trust Calibration**: Over-trust(complacency)=잘못된 조언 무비판 수용 / Under-trust(disuse)=올바른 조언도 무시
- 사용자가 잘못된 AI 결정을 **redirect** 할 수 있어야 함
